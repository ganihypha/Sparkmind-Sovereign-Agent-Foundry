## 1. Prinsip productization

1. **1 aset → banyak produk (multi-SKU).** Aset yang sama dijual dalam beberapa bentuk:
   *DIY (template/skill)*, *Done-with-you (langganan+support)*, *Done-for-you (jasa)*,
   *Lisensi/white-label*, *Edukasi (course)*.
2. **Nama produk = outcome.** "Toko Online Jalan", bukan "fullstack-cycle skill".
3. **Tangga harga (Good-Better-Best + langganan + jasa).** Tangkap WTP dari Rp 49k s/d puluhan juta.
4. **Setiap produk punya halaman & CTA jelas.** Outcome → fitur → harga → "Mulai / Konsultasi".

## 5. LTV ladder (relationship, bukan transaksi)

```
Tripwire (eBook/mini-kelas 49k)
  → Produk inti (vertikal: setup 199k–299k)
  → Langganan (Care Plan / AI Staff 49k–299k/bln)   ← MRR
  → High-ticket (App Custom / AI Company 1.5–15jt)
  → Partner (reseller / MoR-aaS / franchise)          ← network effect
```

Setiap pelanggan baru bisa naik tangga → **AOV & LTV naik tanpa akuisisi baru**.

---

## 6. Unit economics (kenapa tetap untung besar)

| Variabel | Asumsi |
|---|---|
| Infra | ~Rp 0 (Cloudflare free/edge) → **COGS produk digital ~0** |
| Margin produk/langganan | 85–95% |
| Margin jasa | 60–80% (waktu kerja, sebagian diakselerasi agent) |
| Biaya transaksi | MDR Duitku (QRIS ~0.7%) |
| MoR-aaS | net fee 5–10% gross partner |

> Karena COGS mendekati nol, **tiap rupiah AOV tambahan ≈ profit**. Reposition menaikkan
> AOV (paket+langganan+jasa) → profit naik tanpa menaikkan biaya secara linear.
